## 1. 产品概述

### 1.1 产品定位

**颜值评分** 是一款基于 AI 人脸分析技术的纯前端颜值测评工具类应用。用户无需注册登录，上传一张照片即可在数秒内获得专业的五官分析报告、颜值综合评分、专属颜值类型认定，以及个性化的美妆穿搭建议。

- **应用类型：** 纯前端 Web 应用（无后端，无登录）
- **核心技术：** face-api.js 人脸检测与特征提取
- **访问端口：** 9597
- **设计优先级：** 移动端优先（Mobile First）

### 1.2 目标用户

| 用户群体 | 特征描述 | 核心诉求 |
|---------|---------|---------|
| 主力用户 | 18-28 岁年轻女性 | 了解自身五官特点，获得美妆灵感 |
| 次要用户 | 28-40 岁女性 | 寻找适合自己的妆容风格 |
| 扩展用户 | 对自身外貌好奇的男性用户 | 趣味性颜值测评 |

**用户画像（核心）：**
- 活跃于小红书、抖音、微博等社交平台
- 对美妆、穿搭、颜值话题感兴趣
- 喜欢分享有趣的自我测评结果
- 使用手机作为主要上网设备

### 1.3 核心价值

| 维度 | 价值描述 |
|-----|---------|
| 即时反馈 | 3 秒内完成分析，无需等待服务器响应 |
| 专业感 | 多维度评分体系，结果有科学依据感 |
| 趣味性 | 有趣的颜值类型标签，激发分享欲望 |
| 实用性 | 基于五官特点的个性化美妆建议 |
| 隐私安全 | 纯前端处理，照片不上传服务器 |

### 1.4 产品目标

- **体验目标：** 用户从打开页面到获得完整报告，操作步骤不超过 3 步
- **性能目标：** 分析结果在 5 秒内呈现
- **传播目标：** 报告卡片适合截图分享，具备社交传播力

## 2. 功能需求

### 2.1 照片上传

#### 2.1.1 功能描述
用户通过多种方式提供待分析的人脸照片，系统进行预处理后进入分析流程。

#### 2.1.2 上传方式

**方式一：拖拽上传（桌面端）**
- 在上传区域内拖拽图片文件即可触发上传
- 拖拽进入区域时，上传区高亮显示（边框变色、区域半透明遮罩）
- 支持单文件拖入，多文件拖入时取第一张
- 拖拽过程中展示放手提示文案："松开即可上传"

**方式二：点击选择（通用）**
- 点击上传区域弹出系统文件选择器
- 支持格式：JPG、JPEG、PNG、WEBP、HEIC
- 文件大小限制：最大 10MB
- 选择后立即预览并触发分析

**方式三：移动端拍照**
- 移动端显示"拍照"和"从相册选择"两个入口
- 拍照模式默认使用后置摄像头
- 支持 iOS Safari 和 Android Chrome 调用系统相机
- 拍照完成后自动进入分析流程

#### 2.1.3 图片预处理
- 超出 1920×1920 分辨率的图片自动等比压缩
- 对图片进行方向校正（处理 EXIF 旋转信息）
- 预览时在上传区域显示图片缩略图

#### 2.1.4 异常处理
| 异常场景 | 处理方式 |
|---------|---------|
| 文件格式不支持 | 提示"请上传 JPG、PNG 格式的图片" |
| 文件超出大小限制 | 提示"图片大小不能超过 10MB" |
| 图片损坏无法读取 | 提示"图片读取失败，请更换图片" |

### 2.2 人脸检测

#### 2.2.1 功能描述
利用 face-api.js 对上传图片进行人脸检测，确认图片中存在有效人脸后再进行后续评分分析。

#### 2.2.2 检测逻辑
1. 图片加载完成后，自动调用 face-api.js 的人脸检测模型
2. 检测是否存在人脸，并提取人脸特征点（68点模型）
3. 检测置信度阈值：≥ 0.7 视为有效人脸

#### 2.2.3 检测结果处理

**检测成功：**
- 在图片上用半透明矩形框标注检测到的人脸区域
- 多张人脸时，取面积最大的人脸进行分析
- 无缝过渡到评分分析阶段

**检测失败（无人脸）：**
- 显示友好提示弹窗，包含以下内容：
  - 提示图标（相机/人像图标）
  - 主文案："未检测到人脸"
  - 副文案："请上传包含清晰正面人脸的照片，建议人脸占画面 1/3 以上"
  - 操作按钮："重新上传"
- 提供示例说明（什么样的照片效果最好）

**人脸过小：**
- 人脸区域面积小于图片面积的 5% 时提示："人脸图像太小，建议上传更清晰的近距离照片，以获得更准确的评分结果"
- 用户可选择继续分析或重新上传

#### 2.2.4 检测中状态
- 显示动画加载进度条（有进度感，分阶段更新文案）
- 加载文案序列：
  1. "正在识别人脸..."
  2. "分析五官比例..."
  3. "计算颜值评分..."
  4. "生成专属报告..."

### 2.3 颜值综合评分

#### 2.3.1 评分体系

**综合评分范围：** 0 - 100 分
**评分呈现方式：** 数字滚动动画 + 环形进度条

| 分数区间 | 评级 | 对应描述 |
|---------|-----|---------|
| 90 - 100 | S+ 绝世 | 万中无一的天赐容颜，令人过目难忘 |
| 80 - 89 | S 倾城 | 五官精致协调，美貌程度远超常人 |
| 70 - 79 | A+ 出众 | 颜值在线，是同龄人中的佼佼者 |
| 60 - 69 | A 亮眼 | 外貌有明显亮点，整体感觉清新好看 |
| 50 - 59 | B+ 可人 | 五官端正，散发出自然亲和的气质 |
| 40 - 49 | B 普通 | 平凡中自有魅力，气质胜于颜值 |
| 30 - 39 | C 潜力 | 有改造空间，妆容和造型可大幅提升 |
| 0 - 29 | C- 个性 | 独特的外貌风格，风格定位更重要 |

#### 2.3.2 评分显示设计
- **大数字展示：** 中央显示分数（字号超大，带动画）
- **环形进度条：** 围绕分数的彩色环形图，颜色随分段变化
- **评级标签：** 分数下方显示对应评级（如 "S 倾城"）
- **星级指示：** 1-5 星视觉化辅助（分数 ÷ 20 = 星数）

#### 2.3.3 分数计算说明
综合评分由各维度加权计算得出（详见第 3 章评分维度设计），并加入适度的随机偏差（±3分以内），使结果更具差异化。

### 2.4 五官分析报告

#### 2.4.1 分析维度
基于 face-api.js 提取的面部特征点，对以下五官进行分项评分与文字描述：

| 五官项目 | 分析内容 |
|---------|---------|
| 眼睛 | 大小、眼距、眼型（单眼皮/双眼皮/内双）、明亮度 |
| 眉毛 | 眉形（平眉/弓形眉/上扬眉/下垂眉）、眉距、浓密度 |
| 鼻子 | 鼻梁高度、鼻翼宽度、挺拔程度 |
| 嘴唇 | 唇形（厚薄、宽窄）、M型峰值、嘴角走势 |
| 脸型 | 脸型分类（瓜子脸/圆脸/方脸/长脸/心形脸/菱形脸） |
| 肤质 | 基于图像亮度和均匀度评估光泽感 |

#### 2.4.2 报告展示形式
- 每个维度单独展示，含：
  - 维度名称图标
  - 单项评分（进度条形式，满分10分）
  - 3-5句专业描述文字
  - 特征标签（如"杏眼"、"卧蚕明显"、"人中适中"）
- 整体以卡片列表形式从上往下展开

### 2.5 颜值类型分类

#### 2.5.1 功能描述
根据五官特征综合判断用户的颜值类型，给出对应的类型名称、特征描述和代表明星，增强结果的趣味性和分享欲。

#### 2.5.2 类型判定规则

- 类型详细设计见第 4 章

### 2.6 专业化妆建议

#### 2.6.1 功能描述
基于五官分析结果，为用户提供针对性的专业化妆建议，覆盖完整妆容流程。

#### 2.6.2 建议维度

**眉形建议**
- 根据脸型和眉骨特征推荐适合的眉形
- 示例："您的脸型偏圆，建议选择微上扬的拱形眉，拉长脸部视觉比例"
- 提供对比图示意（平眉 vs 拱形眉效果）

**眼妆建议**
- 根据眼型推荐眼线画法、眼影色系
- 小眼用户推荐开眼角技巧
- 单眼皮用户推荐眼影渐变手法

**唇妆建议**
- 根据唇形推荐唇色和画唇技巧
- 薄唇推荐适当外扩画法
- 厚唇推荐内扣减法和深色遮盖技巧

**修容建议**
- 根据脸型推荐高光和阴影位置
- 提供"最显瘦"修容方案
- 鼻梁修容（提鼻梁/收鼻翼）建议

**发型建议**
- 根据脸型推荐适合的发型轮廓
- 推荐刘海类型（有刘海/无刘海/斜刘海）
- 推荐发色方向（暖色系/冷色系/染色建议）

**整体风格定位**
- 根据颜值类型给出整体 Look 方向
- 如："您适合走清甜学院风，避免过于成熟浓艳的妆容"

### 2.7 报告分享

#### 2.7.1 功能描述
将分析结果生成精美图片，用户可保存至相册或分享至社交平台。

#### 2.7.2 分享卡片内容
- 应用名称和 Logo

- 综合评分（大字显示）和评级
- 颜值类型标签
- 3个核心特征标签
- 底部水印："由颜值评分生成"

#### 2.7.3 技术实现

- 生成 PNG 格式，分辨率 1080×1920（竖版）或 1080×1080（方形）
- 支持"保存到相册"（触发下载）
- 支持"复制链接"分享（将结果参数编码至 URL，可复现报告）

## 3. 评分维度设计

综合评分由以下 8 个维度加权计算，总权重为 100%。

### 3.1 维度详细说明

| # | 维度名称 | 权重 | 评分范围 | 说明 |
|---|---------|-----|---------|-----|
| 1 | 五官精致度 | 25% | 0-100 | 眼、鼻、口、眉各器官的形态完整度和精细程度 |
| 2 | 黄金比例契合度 | 20% | 0-100 | 面部三庭五眼比例与黄金分割率的吻合程度 |
| 3 | 面部对称性 | 15% | 0-100 | 左右半脸特征点的对称程度，越对称越高分 |
| 4 | 眼神气质感 | 15% | 0-100 | 眼睛大小、眼白比例与整体神采的综合评估 |
| 5 | 肤质光泽度 | 10% | 0-100 | 图像明度均匀性与皮肤质感的视觉评估 |
| 6 | 脸型立体感 | 8% | 0-100 | 下颌轮廓清晰度、颧骨位置与脸型立体程度 |
| 7 | 整体协调性 | 5% | 0-100 | 五官之间比例协调，不突兀、不违和 |
| 8 | 时代审美契合度 | 2% | 0-100 | 与当下主流审美趋势的匹配程度（趣味维度） |

### 3.2 维度说明补充

**五官精致度（权重最高）**
- 衡量标准：眼睛双眼皮清晰度、鼻梁挺拔程度、嘴唇饱满度、眉形整齐度
- 计算方式：从 face-api.js 特征点坐标推算各器官尺寸比例，与标准美型数据库对比评分

**黄金比例契合度**
- 三庭：发际线到眉心、眉心到鼻底、鼻底到下颌各占面部 1/3
- 五眼：面部宽度约等于五个眼睛宽度，两眼内眼角间距约一个眼宽
- 最优解：三庭均等、五眼对称

**面部对称性**
- 取 33 对对称特征点（左右各 33 个），计算平均欧式距离偏差
- 距离偏差越小，对称性越高，得分越高

**眼神气质感（创新维度）**
- 根据眼睛在面部的占比、眼距与瞳距比例综合评估"眼睛的存在感"
- 配合面部整体亮度，评估视觉上的神采感

**时代审美契合度（趣味维度）**
- 轻量权重，主要用于增添报告趣味性
- 评估当前流行的清冷感、甜系、高级感等审美标签的契合程度

### 4.2 各类型详细描述

---

#### Type-01：清冷仙气型 ❄️ Ice Fairy

**类型描述**
拥有一种超凡脱俗的清冷气质，眉眼之间透着一股淡淡的疏离感，让人忍不住多看几眼。不是最浓艳的颜，却是最难忘的脸。

**主要特征**
- 眼型细长，眼神深邃而疏离
- 面部轮廓偏长，颧骨略高
- 肤色偏白或冷调，皮肤质感细腻
- 嘴唇轮廓清晰，嘴角微微下垂带神秘感
- 整体气质偏冷，适合冷白皮造型

**代表明星**
- 刘亦菲（东方仙气鼻祖，清冷天花板）
- 周冬雨（清冷中带着灵动）
- 倪妮（高级清冷，眼神有故事）

**化妆风格**
- 推荐：雾面妆感、冷调眼影（灰紫、烟熏）、裸色哑光唇
- 避免：过于浮夸的亮片眼影、大红唇

---

#### Type-02：甜系邻家型 🍬 Sweet Neighbor

**类型描述**
像隔壁班上最讨喜的女孩，笑起来有小梨涡，眼睛大而明亮，给人强烈的亲近感和安全感。是会让人想保护的那种甜。

**主要特征**
- 大眼睛，眼距适中，双眼皮深且明显
- 脸型圆润偏鹅蛋，下巴圆润
- 嘴巴偏小，唇形丰满有肉感
- 笑容灿烂，颧肌饱满带"苹果肌"
- 整体五官偏小巧精致

**代表明星**
- 赵露思（标准甜系代表，大眼萌感）
- 虞书欣（甜中带飒，是女孩的梦）
- 关晓彤（甜美国民脸）

**化妆风格**
- 推荐：清甜奶油妆、珊瑚橘眼影、豆沙粉唇、腮红打透
- 避免：过于深邃的烟熏感、冷灰色系

---

#### Type-03：御姐女王型 👑 Ice Queen

**类型描述**
一种与生俱来的气场，无需开口便有压制全场的气质。五官立体，眉目间自带权威感，优雅而凌厉，是让人心生仰望的类型。

**主要特征**
- 眉形利落，多为平眉或微下压
- 面部轮廓分明，颧骨高而突出
- 眼型较深邃，单眼皮或深双
- 鼻梁挺拔，鼻翼收窄
- 整体气质成熟，有强烈的存在感

**代表明星**
- 章子怡（国际章，御姐鼻祖）
- 佟丽娅（东方大美人，气场碾压）
- 范冰冰（浓颜御姐天花板）

**化妆风格**
- 推荐：大红唇、猫眼眼线、高光立体修容、深色眼影
- 避免：清淡粉嫩色系、卡姿兰大眼娃娃风

---

#### Type-04：元气少女型 ⚡ Energetic Girl

**类型描述**
永远充满活力，笑起来让整个房间都亮了。五官小巧灵动，像一道阳光，充满朝气的能量型颜值，看到就心情变好。

**主要特征**
- 五官小巧但精致，眼睛水灵灵
- 脸型偏圆，有婴儿肥
- 嘴角天然上扬，笑容易出酒窝
- 肤色健康，偏暖调
- 头发自然蓬松，发量充足

**代表明星**

## 5. 非功能需求

### 5.1 性能要求

| 指标 | 要求 | 说明 |
|-----|-----|-----|
| 首屏加载时间 | ≤ 3秒（4G 网络） | 主要资源优先加载，face-api 模型懒加载 |
| 人脸检测耗时 | ≤ 2秒 | 使用 tiny_face_detector 轻量模型 |
| 报告生成耗时 | ≤ 3秒 | 包含动画时间在内不超过5秒 |
| 分享图片生成 | ≤ 3秒 | html2canvas 渲染时间 |
| 包体积 | ≤ 5MB（gzip后） | 合理分片，face-api 模型文件单独加载 |

### 5.2 兼容性要求

**移动端（优先）**

| 平台 | 版本要求 |
|-----|---------|
| iOS Safari | iOS 13+ |
| Android Chrome | Android 8+ |
| 微信内置浏览器 | 支持（部分功能降级） |

**桌面端**

| 浏览器 | 版本要求 |
|-------|---------|
| Chrome | 80+ |
| Firefox | 75+ |
| Safari | 13+ |
| Edge | 80+ |

### 5.3 安全要求

- **隐私保护：** 所有图像处理在浏览器本地完成，不向任何服务器上传图片
- **数据存储：** 不在本地存储（localStorage/cookie）持久化任何用户照片信息
- **CSP 策略：** 配置合理的内容安全策略，防止 XSS 攻击
- **HTTPS：** 生产环境强制 HTTPS（相机权限需要安全上下文）

### 5.4 可访问性

- 所有图片元素提供 alt 文字描述
- 支持键盘导航（Tab 键焦点可见）
- 文字对比度满足 WCAG AA 级别（对比度 ≥ 4.5:1）
- 分析中状态提供屏幕阅读器可读的进度提示

```
[评分结果页]
  - 综合评分（滚动动画）
  - 颜值类型标签

  - 分享按钮
      |

      |

      |

      |

[返回首页]
```

### 6.2 各页面状态说明

**首页状态**
- 默认状态：展示上传引导
- 拖拽中：上传区高亮（背景色变化 + 边框动画）
- 文件验证失败：Snackbar 提示（底部弹出，3秒自动消失）

**检测中状态**
- 不可交互（上传按钮禁用）
- 进度条动画循环
- 文案每 1.5 秒切换一次

**结果页状态**
- 进入时评分数字从 0 滚动到目标分数（1.5秒）
- 各分析卡片依次淡入（stagger 动画，间隔 0.2 秒）
- 分享按钮常驻底部（fixed）

### 6.3 错误处理流程

| 错误类型 | 展示方式 | 用户操作 |
|---------|---------|---------|
| 格式不支持 | Snackbar 底部提示 | 自动关闭 |
| 文件过大 | Snackbar 底部提示 | 自动关闭 |
| 无人脸检测 | 全屏弹窗 | 点击"重新上传" |
| 人脸过小 | 内嵌提示条 | 继续 or 重新上传 |
| 浏览器不支持 | 全屏提示页 | 建议更换浏览器 |
